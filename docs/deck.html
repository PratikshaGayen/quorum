“We don’t make AI smarter — we make its decisions trustworthy and auditable, by turning one AMD Instinct GPU into a courtroom.”

Problem & Context

A single LLM is a confident, unauditable single point of failure.

  • KYC / onboarding approvals are high-stakes: a wrong APPROVE onboards a fraudster; a wrong REJECT loses a real customer. Both carry regulatory and financial penalties.
  • A solo LLM hallucinates, is overconfident, and leaves no record of why — unacceptable for a regulated compliance decision that must be defensible to an auditor.
  • Our evidence: on 151 cases, a single strong model (Qwen2.5-7B) confidently approved 57% of the bad actors it should have caught (recall 0.43) — with no dissenting view to flag the risk.
  • Stakeholders: compliance & risk teams, auditors / regulators, and the customers wrongly blocked or wrongly onboarded.

  • AI approach: multi-agent consensus, prompt-induced diversity (one model, distinct personas + temperatures), reason-then-decide structured output, RAG-free.
  • Reliability built in: strict JSON parse + retry + ABSTAIN fallback; principled ESCALATE when the panel genuinely splits → ambiguous cases go to a human.
  • Stack: vLLM (ROCm) · FastAPI backend · Next.js dashboard.
  • Built during the hackathon: the whole system — jurors, orchestrator, adjudicator, eval + AMD benchmark, the synthesized dataset, the dashboard. No prior/proprietary code; open-weight models.

Headline — contradiction-catch rate 0.285: in 43 / 151 cases the confident baseline was wrong and the council corrected it.
  • Model: Qwen2.5-7B-Instruct on vLLM (ROCm), 3 personas + baseline.
  • HW: 1 × MI300X, ~165 / 192 GiB VRAM (~86%) — whole jury on one card.
  • Data: 151 synthesized, labeled KYC cases (70 APPROVE / 81 REJECT), fully synthetic & non-proprietary.
  • AMD concurrency: jury in parallel vs one-at-a-time → 56.9 vs 36.7 decisions/min (1.55×); p95 1,377 vs 2,050 ms.
  • ESCALATE: 29 / 151 (~19%) routed to a human — only on genuine REJECT-risk cases, never on clean ones.
Disclosure: dataset is synthetic (method in repo). Council accuracy counts an ESCALATE as “correct” (avoided a wrong auto-decision). Model-independent claims: recall 0.43→0.96, catch-rate 0.285.

Impact & Demo Summary

The diversity + dissent — not a bigger model — is what makes the decision trustworthy.

Impact

  • Fewer fraudulent approvals (recall 0.43 → 0.96) with no increase in false rejections (precision stays 1.0).
  • Auditable by construction: every decision ships a confidence score + a verbatim dissent report — the “why” a regulator demands.
  • Right-sizes human effort: auto-decides the clear cases, escalates only the genuinely ambiguous ~19%.

Future work

  • Multi-model council (the 192 GB MI300X easily holds 3+ models); an LLM chief-judge adjudicator; more / specialized jurors; real labeled case data.

Demo flow (~60s)

  • Open the dashboard on the planted case DEMO-FRAUD — reads clean and professional.
  • Single model → APPROVE, high confidence. “A solo LLM waves a layering pattern through.”
  • The jury → splits; compliance officer & devil’s advocate flag the pass-through / income mismatch → adjudicator returns ESCALATE with the dissent on record.
  • “The solo model would have onboarded it. The council caught it and produced the audit trail — on one AMD GPU, in parallel.”
